# 质量分析管理系统项目说明

## 1. 项目简介

本项目是“中储粮镇江直属库有限公司质量分析管理系统”的前端演示应用，面向粮食质量检验全生命周期管理场景。系统以质量看板为入口，覆盖扦样、送样、收样、样品标签、检验结果录入、逐级审批、留样/销样、质量预警、质量报表、检化验室设备与试剂管理等业务。

当前为纯前端 Demo：业务初始数据内置于代码中，变更数据通过浏览器 `localStorage` 保存，未接入业务后端、数据库或真实设备。

## 2. 技术栈与运行方式

- 框架：Vue 3（Composition API、`<script setup>`）
- 语言：TypeScript
- 构建工具：Vite 5
- 路由：Vue Router 4
- UI：全局 CSS 为主，Tailwind CSS 已配置
- 图标：lucide-vue-next
- 工程检查：ESLint、vue-tsc

常用命令：

```bash
npm install
npm run dev
npm run check
npm run lint
npm run build
```

其中：

- `npm run dev`：启动开发服务器。
- `npm run check`：执行 Vue / TypeScript 类型检查。
- `npm run lint`：执行 ESLint 检查。
- `npm run build`：先执行类型检查，再构建生产包。

```text
质量管理/

│  ├─ composables/useTheme.ts 主题组合式函数（当前未接入）
│  └─ lib/utils.ts            cn 工具函数（当前未接入）
```

应用启动链路：`index.html` 挂载 `#app` → `src/main.ts` 创建 Vue 应用并安装路由 → `App.vue` 渲染路由出口 → 路由 `/` 加载 `HomePage.vue`。

当前仅配置根路径 `/`。侧栏菜单通过页面内 `activeMenu` 状态切换内容，不改变 URL。

## 4. 核心业务模块

### 4.1 登录与角色

系统提供前端内存登录和角色演示。角色用于控制报告、试剂领用审批节点：

| 用户名 | 密码 | 姓名 | 角色 |
| --- | --- | --- | --- |
| wangshuai | 123456 | 王帅 | 编制人 |
| lishenhe | 123456 | 李审核 | 审核人 |
| zhaopizhun | 123456 | 赵批准 | 批准人 |
| zhijianyuan | 123456 | 质检员A | 质检员 |

登录状态仅保存在当前页面内存中；退出或刷新后需要重新登录。账号与密码均为演示数据，不能作为生产认证方案。

### 4.2 质量看板

- 展示库存、仓房、品种、合格率、风险仓房、待检批次等综合指标。
- 基于库区底图展示仓房质量点位与风险等级。
- 汇总正常、一级、二级、三级预警，并支持查看不合格项。
- 支持仓房点位拖动校准、恢复默认和本地保存。

### 4.3 检验流程管理

该模块包含以下子功能：

1. **流程总览**：展示扦样、待检、待审、留样、销样等统计及流程节点。
2. **扦样单管理**：支持内部/外部扦样单新增、编辑、提交、查看、送样、收样；录入中记录可删除。
3. **样品标签管理**：已收样样品自动生成标签记录，支持状态维护、标签打印与隐藏删除。
4. **检验报告管理**：已收样样品自动进入报告列表，通过扫码弹窗核对样品后进入结果录入；可勾选检化验仪器自动获取并填充对应检测数据，再进入三级审批；审批通过后生成正式质检报告，支持打印和 CSV 导出。
5. **样品台账管理**：基于收样记录生成台账，并提供粮种、来源、事由和日期筛选。
6. **样品留样管理**：展示审批通过后转入留样的样品及到期状态。
7. **销样管理**：展示审批不同意转销样、以及留样到期的样品。

样品主流程：

```text
录入中 → 扦样完成 → 已送样 → 已收样
                         ↓
                 待检 → 在检 → 检毕
                         ↓
          编制人审批 → 审核人审批 → 批准人审批
                         ↓
                     留样 / 销样
```

### 4.4 检验标准、结果判定与审批

- 系统内置玉米、大豆的入库验收、出库检测标准。
- 春季普检、秋季普检复用出库检测指标。
- 检测值录入后，依据 `≤`、`≥`、`<`、`>` 和“正常”等标准自动判定合格或不合格。
- 报告审批采用“编制人 → 审核人 → 批准人”三级流转；仅当前角色与审批节点匹配时可操作。
- 审核/批准可退回修改；批准人可不同意并转销样；批准通过后样品进入留样。

### 4.5 质量预警与报表

质量预警从检验报告不合格项动态计算：

- 一级预警：关键不合格至少 2 项，或 1 项关键不合格且普通不合格至少 2 项。
- 二级预警：普通不合格至少 3 项，或 1 项关键不合格且普通不合格不超过 1 项。
- 三级预警：无关键不合格，普通不合格 1 至 2 项。

报表模块动态整理已完成的报告，可查看并打印：

- 质量分析货位明细表
- 分货位质量汇总表
- 质量分析验收申请函
- 春秋普检测结果汇总表

### 4.6 检化验室管理

- 仪器类别与仪器台账维护。
- 仪器检定/校准计划、历史记录与维护保养登记。
- 实验室温湿度模拟历史和趋势展示。
- 仪器新增、类别新增、维护、弃用等修改仅在当前页面内存中保留，刷新后恢复初始数据。

### 4.7 检化验试剂管理

- 管理普通试剂与易制毒试剂的库存、阈值、存放位置、保管人和状态。
- 支持试剂入库、领用、归还、盘点等过程留痕。
- 领用记录经过“编制人 → 审核人 → 批准人”审批；批准后才扣减库存。
- 归还会回补库存并自动生成待盘点记录。
- 展示试剂柜温湿度模拟趋势。

### 4.8 用户管理

展示并维护演示用户信息、角色、部门、联系电话和启用状态。用户数据属于本地持久化数据。

## 5. 数据结构与状态管理

项目未使用 Pinia 或 Vuex，页面状态全部集中于 `HomePage.vue` 的 `ref`、`computed` 和 `watch`。

主要数据实体：

| 实体 | 说明 |
| --- | --- |
| `SamplingRecord` | 样品核心实体，贯穿扦样、收样、标签、报告、审批、台账、留样与销样。 |
| `SamplingForm` | 扦样录入表单。 |
| `ReportMeta` / `ReportResult` | 报告元数据与单项检测结果。 |
| `QualityStandard` | 粮种和检验事由对应的检测标准。 |
| `SystemUser` | 演示账号、登录与审批角色。 |
| `LabInstrument` | 仪器台账、检定、维护信息。 |
| `ReagentStock` / `ReagentFlowRecord` | 试剂库存及入库、领用、归还、盘点过程记录。 |
| `EnvironmentRecord` / `ReagentCabinetEnvironment` | 实验室及试剂柜温湿度记录。 |

## 6. 数据来源与持久化

### 初始数据

样品、质量标准、预警演示记录、仪器、环境、试剂、用户等数据均为前端 Mock 数据，集中定义在 `src/pages/HomePage.vue`。

### localStorage

业务主状态保存在：

```text
quality-management-grain-standard-state-v2
```

持久化范围：

- 内部、外部扦样记录
- 用户列表
- 被删除的标签编号
- 试剂库存
- 试剂过程记录

仓房地图点位校准单独使用：

```text
quality-management-warehouse-map-calibration-v1
```

仪器、仪器类别、检定记录、维护状态、试剂柜环境、当前登录状态、弹窗及筛选状态不持久化。

## 7. 静态资源与外部依赖

`public` 中包含登录背景、企业 Logo、库区地图、favicon 及粮食质量指标参考表等静态资源。

项目没有业务后端 API、Axios 或数据库依赖。运行时唯一外部服务为样品标签二维码图片服务：`https://api.qrserver.com`。

## 8. 当前实现边界

- 当前系统定位为可视化前端演示，非生产系统。
- 所有业务逻辑集中在单个 `HomePage.vue` 文件中，尚未按领域拆分组件、Store、API 和数据模块。
- 路由仅有根页面，菜单不具备独立 URL、刷新恢复、深链接与路由权限守卫能力。
- 登录、权限、审批及数据持久化均为前端模拟，不具备安全认证、服务端鉴权、审计与多端一致性能力。
- 实验室和试剂柜环境曲线为模拟生成，未接入真实设备采集。

如需进入生产化开发，建议优先拆分领域页面与组件，再建设 API 层、服务端认证授权、数据库模型、真实设备数据接入和审计日志。
